# Import Article To Blog

用于把外部文章转成这个博客能直接使用的标准文章。

## 目标

- 读取用户提供的 `markdown / docx / doc / txt / pdf`
- 整理正文结构，转成博客可用 Markdown
- 补齐 front matter
- 把文章放到 `source/_posts/`
- 尽量只使用仓库中已有的分类和 tags

## 固定约束

- 文章主文件放在 `source/_posts/<标题>.md`
- 如果文章需要本地封面或资源目录，则使用同名目录：
  - `source/_posts/<标题>/cover.webp`
- `cover` 字段优先写 `cover.webp`
- front matter 基本模板：

```yaml
---
title: 标题
author: 作者
date: 2026-04-10 00:00:00
categories:
  - 引力波
tags:
  - 社论
cover: cover.webp
---
```

## 分类规则

当前仓库只稳定使用两个分类：

- `幻协生存指南`
- `引力波`

选择规则：

- 如果文章是社团管理、招新、摆摊、活动、外联、会刊、征文、影展、观影、经验总结这类指南，使用 `幻协生存指南`
- 其他一般文章默认使用 `引力波`

不要新增新的 `categories`，除非用户明确要求。

## Tag 规则

优先只从现有 tag 中选：

- `科幻小说`
- `幻协生存指南`
- `社团资料`
- `世界科幻`
- `社论`
- `翻译`
- `杂谈`
- `社团管理`
- `组织运营`

选择规则：

- 指南类文章：
  - 至少包含 `幻协生存指南`
  - 一般再加 `社团资料`
  - 如果文章主题明确涉及管理，可再加 `社团管理`
  - 如果文章明显偏组织协作或制度，可再加 `组织运营`
- 翻译文章：
  - `翻译`
  - 若内容是海外/国际科幻信息，可再加 `世界科幻`
- 评论、论述、专题分析：
  - 优先 `社论`
  - 轻量感想可用 `杂谈`
- 小说：
  - `科幻小说`

除非用户明确要求，不要发明新 tag。

需要快速查看当前词表时，读 `references/taxonomy.md`。

## 来源格式处理

### Markdown

- 保留正文结构
- 修正标题层级、图片路径、脚注、引用块
- 删除不适合直接放正文的零散作者信息、投稿说明、页眉页脚

### docx / doc

- 优先用 `pandoc` 或可用的 doc/docx 处理工具抽取正文
- 转成 Markdown 后再人工整理标题、列表、图片、脚注
- 如果有单独的作者行、学院行、比赛信息，默认不放进正文首段，除非它本身属于文章内容

### txt

- 按自然段切分
- 手工补标题、引用、二级标题

### pdf

- 先抽取文本
- 清理分页残留、断行、页码、脚注串位
- 如果版式复杂导致抽取混乱，要优先保证正文可读性，而不是机械保留版面

## 标准化步骤

1. 读取源文件并抽取纯正文
2. 确定文章标题、作者、日期
3. 判断分类：`引力波` 或 `幻协生存指南`
4. 从现有 tag 集合里选最贴近的 1-3 个
5. 写入标准 front matter
6. 修正正文格式
7. 如需封面：
   - 指南类文章优先使用 `.codex/skills/add-guide-article/scripts/generate-guide-cover.sh`
8. 构建验证：

```bash
pnpm run clean
pnpm run build
```

## 与其他 skills 的关系

- 如果是“幻协生存指南”系列文章，同时使用 `add-guide-article`
- 如果需要批量整理旧封面格式，可再使用 `convert-post-covers`
